# 数据说明：绿色转型试点政策评估

本文件夹（`01_data/`）存放三张原始数据表，用于评估「绿色转型试点」政策对城市绿色创新的影响。三张表来自不同渠道、由不同口径整理而来，**合并前需要先查验、对齐**。

## 研究背景

某国自 2016 年起，分三批（2016、2018、2020 年）在部分城市推行「绿色转型试点」政策。我们关心：试点是否提升了城市的绿色创新水平（以**每万人绿色专利授权量**衡量），效应随时间如何演化，不同批次之间有无差异。样本期为 **2012–2024 年**。

## 三张数据表

### 1. `city_economic.csv`——城市经济面板

- **模拟来源**：统计年鉴
- **内容**：覆盖全部城市 × 全部年份的经济面板

| 字段 | 含义 | 类型 / 单位 |
|------|------|------|
| `city_code` | 城市代码（唯一数字标识） | 整数 |
| `city_name` | 城市名称 | 文本 |
| `year` | 年份 | 整数（2012–2024） |
| `gdp` | 地区生产总值 | 亿元 |
| `pop` | 常住人口 | 万人 |
| `second_share` | 第二产业占比 | 百分数（%） |
| `region` | 所在区域 | 文本（东部 / 中部 / 西部） |

> `region` 是**文字型分类变量**。若要在模型中用它（如作分组/异质性），通常需要先把它**编码成数字并建立一张"文字↔数字"对应表**。

> 这是三张表中**唯一同时包含 `city_code` 与 `city_name` 的表**，因此它是把"城市名"与"城市代码"对应起来的**桥梁**。

### 2. `policy_pilot.csv`——试点名单

- **模拟来源**：政策文件手工整理
- **内容**：仅包含**试点城市**，记录每个试点城市的批次与起始年份

| 字段 | 含义 | 类型 |
|------|------|------|
| `city_name` | 试点城市名称 | 文本 |
| `batch` | 试点批次（1 / 2 / 3） | 整数 |
| `pilot_year` | 首次试点年份（2016 / 2018 / 2020） | 整数 |

### 3. `city_green.csv`——绿色专利产出

- **模拟来源**：专利数据库
- **内容**：城市 × 年份的绿色创新产出（结果变量）

| 字段 | 含义 | 类型 / 单位 |
|------|------|------|
| `city_name` | 城市名称 | 文本 |
| `year` | 年份 | 整数 |
| `green_patents` | 每万人绿色专利授权量 | 件 / 万人 |

## 三张表的关系

三张表最终要合并成一张「城市-年份」面板：以 `city_economic` 为主表（它提供 `city_code` 和经济协变量），把另外两张表挂接上去。

```text
                     ┌──────────────────────────┐
  policy_pilot ──────┤  city_economic（主表）    ├────── city_green
   按 city_name 对齐  │  含 city_code + 协变量    │  按 city_name + year 对齐
   （标记试点批次与年）└──────────────────────────┘  （提供 green_patents）
```

- `city_economic`：完整面板，提供 `city_code` 与 `gdp / pop / second_share` 等协变量；
- `policy_pilot`：用 `city_name` 关联，标记哪些城市、从哪一年起成为试点（据此构造处理变量）；
- `city_green`：用 `city_name` + `year` 关联，提供结果变量 `green_patents`。

## 使用提示

三张表来源不同、整理口径不一，**合并前请务必查验**：城市名在各表间是否一致、主键（城市 / 城市-年份）是否唯一、是否存在重复或缺失记录、试点名单里的城市能否全部匹配到经济面板。发现的问题及其处理规则，请在分析报告中如实记录。
