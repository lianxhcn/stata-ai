# 数据说明（练习用·"脏"数据版）

::: 提示
这是一份**专门用来练习"数据查验与清洗"的脏数据**。它的研究场景、字段和表关系与主 demo（`demo/01_data/`）完全一致，但在里面**混入了多种常见的数据质量问题**——请把它当作真实世界里"刚拿到手、还没整理"的数据来对待：**先仔细查验、把发现的问题列出来，再决定怎么处理。**
:::

## 研究背景

某国自 2016 年起，分三批（2016、2018、2020 年）在部分城市推行「绿色转型试点」政策。我们关心：试点是否提升了城市的绿色创新水平（以**每万人绿色专利授权量**衡量），效应随时间如何演化，不同批次之间有无差异。样本期为 **2012–2024 年**。

## 三张数据表（应有的字段）

### 1. `city_economic.csv`——城市经济面板（来源：统计年鉴）

| 字段 | 含义 | 应为的类型 / 单位 |
|------|------|------|
| `city_code` | 城市代码（唯一数字标识） | 整数 |
| `city_name` | 城市名称 | 文本 |
| `year` | 年份 | 整数（2012–2024） |
| `gdp` | 地区生产总值 | 亿元（数值） |
| `pop` | 常住人口 | 万人（数值） |
| `second_share` | 第二产业占比 | 百分数（数值） |
| `region` | 所在区域 | 文本（东部 / 中部 / 西部），用于分组时需编码为数字 |

> 这是三张表中**唯一同时包含 `city_code` 与 `city_name` 的表**，是"城市名 ↔ 城市代码"的桥梁。

### 2. `policy_pilot.csv`——试点名单（来源：政策文件手工整理）

| 字段 | 含义 | 类型 |
|------|------|------|
| `city_name` | 试点城市名称 | 文本 |
| `batch` | 试点批次（1 / 2 / 3） | 整数 |
| `pilot_year` | 首次试点年份（2016 / 2018 / 2020） | 整数 |

### 3. `city_green.csv`——绿色专利产出（来源：专利数据库）

| 字段 | 含义 | 应为的类型 / 单位 |
|------|------|------|
| `city_name` | 城市名称 | 文本 |
| `year` | 年份 | 整数 |
| `green_patents` | 每万人绿色专利授权量 | 件 / 万人（数值） |

## 三张表的关系

以 `city_economic` 为主表（提供 `city_code` 与协变量），把 `policy_pilot`（按 `city_name`）和 `city_green`（按 `city_name` + `year`）挂接上去，最终合并成一张「城市-年份」面板。

## 使用提示（这份数据比较"脏"）

拿到数据别急着分析。**建议先做一遍彻底的数据体检**，重点看：

- **字段类型**：本该是数值的列，是不是被当成文本读进来了？为什么？
- **缺失值**：除了空白，有没有用 `NA`、`.`、`-` 等**非标准记号**表示的缺失？
- **离群 / 异常值**：有没有大得离谱、或不该出现（负数、0）的取值？
- **重复记录**：有没有完全重复的行？
- **跨表一致性**：同一个城市在三张表里的名字写法一致吗？试点名单里的城市，能全部匹配到经济面板吗？
- **文字变量**：像 `region` 这样的分类文本，若要进模型，需不需要先编码成数字、并留一张对应表？

把发现的问题、以及你打算怎么处理，都记录下来——这正是真实研究里最花时间、也最能体现判断力的一环。
