📌 课程导览

本课主线

Stata 的角色→nbstata → Notebook→对话式 AI → Plan→Agent → 本地执行→人：监理

Stata、Python、R 都是这条主线上可以替换的零件。想清楚要解决什么问题，再决定用哪种工具、交给谁做、如何检查——这比纠结”学哪个软件”更重要。

本课要点：你会看到什么

  • Stata 最小入口——在 AI 时代，Stata 该掌握的”最小必要”：界面、dofile、工作目录、帮助文件、外部命令。（第 1 讲）
  • VS Code + nbstata + Copilot——把笔记、代码、结果放进同一个 notebook，并让 Copilot 帮你补全代码、生成注释。（第 2 讲）
  • 对话式 AI 与执行型 Agent——ChatGPT / Claude 负责拆解规划，Codex / Claude Code 负责在本地项目里执行；关键中间物是一份”任务说明书”。（第 3 讲）
  • Agent 演示——在一个 demo/ 项目里跑一遍”计划 → 确认 → 执行 → 检查”，并看一张 TWFE vs 现代 DID vs 真值的对照表。（第 4 讲）

按需选择你的路径

不同学员需求不同，可以按下面的路径各取所需：

  • 只想快速把 Stata 用起来 → 看 第 1 讲，配合 环境配置 的第 1 节（Stata 与外部命令）。
  • 更关心 AI 辅助的实证分析流程 → 看 第 3 讲、第 4 讲 和 附录 A（任务说明书与提示词模板），并下载 demo/ 亲手复现本课演示。
  • 想把本地环境完整配好 → 通读 环境配置，并完成下面「课后务必完成本地配置」。
  • 想系统学习实证研究 → 欢迎报名 7 月 21–23 日的连享会 2026 暑期班·初级班（详见 前言）。

课后务必完成本地配置

受课时所限，安装与配置的完整过程更适合课后按文档从容完成。但请注意：在本地用 Agent 和 AI 工具做分析，前提就是先把环境配好。 因此，请务必花点时间，按下面的文档把配置走通。

好消息是：配置过程本身就可以借助网页版 ChatGPT 或 Claude 边问边做——把你的系统信息、报错原文贴给它，让它一步步指导你。这本身也是本课”用 AI 辅助”理念的一次练习。

Stata 安装的两个要点

Stata 本体的安装很简单，通过正版渠道下载安装即可。只需特别注意两点：

  1. 安装路径不要包含中文字符，也不要有空格等特殊字符（例如用 D:\stata19 这样的纯英文路径）。
  2. 安装完成后，确认安装目录下存在 STATA.LIC 文件——这是 nbstata 插件，以及 Agent、MCP 等工具能够正确关联到你的 Stata 应用程序的关键。

以上两点及更完整的说明，见 ds2026 · 配置 Stata 环境（nbstata）。
