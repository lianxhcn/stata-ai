# T4：彩排即预跑（人主导，全程录屏）

> 这一步一石三鸟：①明晚课堂直播的完整彩排；②意外时的备份录像；
> ③仓库里的预跑成果（output/report 留给听众对照）。

## 0. 起跑前检查（约 5 分钟）

1. 在 demo/ 目录用 Stata 跑 `_dgp/check_packages.do`，按提示装齐缺失命令
   （核心依赖：reghdfe、ftools、csdid、drdid、estout；jwdid 可选）。
2. 终端确认 `quarto --version` ≥ 1.4（typst 引擎生成 PDF 需要）。
3. **打开录屏**（整个 T4 全程录制，这是明晚的保险）。
4. 新开终端，`cd demo`，启动 claude。

## 1. 启动提示词（原样粘贴）

```
请先阅读本文件夹中的 README.md。约束：只允许使用本文件夹（demo/）内的文件，
不得读取 _dgp 文件夹，也不得读取上级目录中的 working、_materials、_task 等内容。

第一步：只输出一份研究计划，包括——数据合并与查验方案、估计策略、
将要创建的文件清单、最终报告的结构。不要执行任何代码，等待我确认后再动手。
```

## 2. 审阅计划（课堂上这就是"人定计划、机器干活"的高潮段）

对照下面清单看计划是否齐备，缺什么就口头补要求（这个来回本身就是最好的演示素材）：

- [ ] 发现三表主键不一致，提出 name–code 桥接方案（经由 city_economic）
- [ ] 有明确的数据查验步骤：重复、缺失、名称不一致/改名
- [ ] 估计策略：TWFE（reghdfe）作基准 + csdid + jwdid/lwdid + 事件研究图
- [ ] 标准误在城市层面聚类
- [ ] 主控脚本 run.do + 分步 dofile + logs/ 留痕
- [ ] report.qmd → PDF（typst），结构含 数据/方法/结果/结论

确认话术建议：「计划可以，补充两点：第一……第二……。确认后开始执行，
每完成一个阶段简要汇报。」

## 3. 执行阶段（agent 跑，你准备课堂话术）

课堂上这几分钟正好回头讲「任务说明书怎么写」（附录 A 页面）。彩排时给自己计时：
从确认到 PDF 产出的总时长，决定明晚直播切到哪一步、其余用预跑成果。

若中途报错：让 agent 自己读 log 修复；重复卡死超过两轮就记录问题、简化要求
（例如跳过 jwdid），保证今晚必须跑完一整轮。

## 4. 收尾核对（约 10 分钟）

1. 打开 `04_report/report.pdf`，快速通读：表格是否成形、事件研究图是否正常。
2. **对照真值**：打开 `working/dgp_truth.md`，让 agent 生成
   `03_output/truth_comparison.md`——把 TWFE、csdid、jwdid 的整体 ATT 与真值并排列表。
   预期图景：TWFE 明显偏离，新方法落在真值附近。这张表是明晚的点睛之笔。
   （给 agent 真值时直接把数值贴进对话即可，仍不让它读 working/。）
3. 确认 02_code、03_output、04_report、logs 分类归档完整——这就是
   「可提交编辑部」的样子，课堂上要展示这个文件夹全貌。
4. 停止录屏，检查录像文件可回放，另存一份到仓库外的安全位置。

## 5. 课堂使用方案速记

- 直播段（约 8 分钟）：启动提示词 → 计划 → 你补两点要求 → 确认开跑；
- 跑动期间：切 slides 讲任务说明书写法；
- 展示段：切回预跑成果——文件夹全貌 → report.pdf → truth_comparison 对照表；
- 兜底：任何环节出事，切 T4 录屏。
